Token Monitor 管理画面

Claude Code トークン監視アプリの接続先サーバーとパラメーターを設定します。
設定はサーバー側に保存されます。

サーバー接続 必須
Token Monitor サーバーのアドレス (例: http://192.168.1.10:3000)
Anthropic API 設定 必須
console.anthropic.com > Settings > API Keys から取得
通常は変更不要
レート制限ヘッダー取得用の軽量リクエストに使用するモデル
Slack 通知設定 任意
Slack App > Incoming Webhooks から取得 (下記の手順を参照)
監視設定 任意
残量がこの値以下で警告通知

パラメーター取得方法

Anthropic API Key の取得
  1. console.anthropic.com にログイン
  2. 左メニュー「Settings」→「API Keys」を開く
  3. 「Create Key」をクリック
  4. キー名を入力して作成 → 表示されるキーをコピー

キーは sk-ant-... で始まる文字列です。
作成後は再表示できないため、すぐに保存してください。

Slack Incoming Webhook URL の取得
  1. api.slack.com/apps にアクセス
  2. 「Create New App」→「From scratch」を選択
  3. App名 (例: Token Monitor) とワークスペースを入力して作成
  4. 左メニュー「Incoming Webhooks」→ トグルを「On」に
  5. 「Add New Webhook to Workspace」をクリック
  6. 通知先チャンネルを選択して「Allow」
  7. 表示される Webhook URL をコピー

URLは https://hooks.slack.com/services/T.../B.../... の形式です。

監視サーバーの起動方法
  1. リポジトリをクローン: git clone ...
  2. cd token-monitor
  3. pip install -r requirements.txt
  4. python app.py

サーバーが http://localhost:3000 で起動します。
この管理画面からサーバーURLを指定して設定を保存すると、
環境変数や .env ファイルなしで動作します。
